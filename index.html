git config

Name usage:

git config –global user.name “[name]”

Email usage:

git config –global user.email “[email address]”

This command sets the author name and email address respectively to be used with your commits.

username@branch:~$ git config --global user.name "[name]"
username@branch:~$ git config --global user.email "[email]"
git init

Usage:

git init [repository name]

This command is used to start a new repository.

username@branch:~$ git init /home/user/Documents/Repos
Initialized empty Git repository in /home/edureka/Documents/Demo/.git/
git clone

Usage:

git clone [url]

This command is used to obtain a repository from an existing URL.

username@branch:~$ git clone https://github.com/mdnzi/FCC-Technical-Documentation-Page.git
Cloning into 'TechnicalDocumentationPage'

Unpacking objects: 100% (8/8), done.
git add

Usage:

git add [file]

This command adds a file to the staging area.

username@branch:~/Documents/Repos$ git add index.html

Usage:

git add *

This command adds one or more to the staging area.

username@branch:~/Documents/Repos$ git add *
git commit

Usage:

git commit -m “[ Type in the commit message]”

This command records or snapshots the file permanently in the version history.

username@branch:~/Documents/Repos git commit -m "Initial Commit"
[master (root-commit) aff3269] First Commit
2 files changed, 300 insertions(+)
create mode 100644 TechnicalDocumentationPage/index.html
create mode 100644 TechnicalDocumentationPage/style/main.css

Usage:

git commit -a

This command commits any files you’ve added with the git add command and also commits any files you’ve changed since then.

username@branch:~/Documents/Repos$ git commit -a
On branch master
nothing to commit, working tree clean

git reset

Usage:

git reset [file]

This command unstages the file, but it preserves the file contents.

username@branch:~/Documents/Repos/TechnicalDocumentationPage/style$ git reset main.css
Unstaged changes after reset:
M TechnicalDocumentationPage/style/main.css
M TechnicalDocumentationPage/index.html

Usage:

git reset [commit]

This command undoes all the commits after the specified commit and preserves the changes locally.

username@branch:~/Documents/Repos$ git reset 09bb8e3f996eaf9a68ac5ba9d9b9fceb0e8541e7
Unstaged changes after reset:
M TechnicalDocumentationPage/style/main.css M TechnicalDocumentationPage/index.html

Usage:

git reset –hard [commit]

This command discards all history and goes back to the specified commit.

username@branch:~/Documents/Repos$ git reset --hard b01557d80d5f53dcf0ebdde4d3f8b0d20db9b9c16
HEAD is not at b01557d Changes made in HTML file
git status

Usage:

git status

This command lists all the files that have to be committed.

username@branch:~/Documents/DEMOS$ git status
On branch master
Changes not staged for commit:
(use "git add ..." to update what will be commited)
(use "git checkout -- ..." to discard changes in working directory)
modified: TechnicalDocumentationPage/style/main.css
modified: TechnicalDocumentationPage/index.html
no changes added to commit (use "git add" and/or "git commit -a")
git rm

Usage:

git rm [file]

This command deletes the file from your working directory and stages the deletion.

username@branch:~/Documents/Repos/TechnicalDocumentationPage$ git rm example.txt
rm 'TechnicalDocumentationPage/example.txt'
git tag

Usage:

Usage: git tag [commitID]

This command is used to give tags to the specified commit.

username@branch:~/Documents/Repos$ git tag b01557d80d5f3dcf0ebdde4d3fb0d20d9b9c16
username@branch:~/Documents/Repos$ git tag
a826d86f183838ae59775fe5a91cbe9023ff3d
b01557d80d5f3dcf0ebdde4d3fb0d20d9b9c16
git branch

Usage:

git branch

This command lists all the local branches in the current repository.

username@branch:~/Documents/Repos$ git branch
* master

Usage:

git branch [branch name]

This command creates a new branch.

username@branch:~/Documents/Repos$ git branch branch_1

Usage:

git branch -d [branch name]

This command deletes the feature branch.

username@branch:~/Documents/Repos$ git branch -d branch_1
Deleted branch branch_1 (was be040cc).
git checkout

Usage:

git checkout [branch name]

This command is used to switch from one branch to another.

username@branch:~/Documents/Repos$ git checkout branch_2
Switched to branch 'branch_2'

Usage:

git checkout -b [branch name]

This command creates a new branch and also switches to it.

username@branch:~/Documents/Repos$ git checkout -b branch_4
Switched to a new branch 'branch_4'
git merge

Usage:

git merge [branch name]

This command merges the specified branch’s history into the current branch.

username@branch:~/Documents/Repos$ git merge branch_2
Merge made by the 'recursive' strategy
TechnicalDocumentationPage/index.html | 2+-
1 file changed, 1 insertion(+), 1 deletion (-)
git remote

Usage:

git remote add [variable name][remote server link]

This command is used to connect your local repository to the remote server.

username@branch:~/Documents/Repos$ git remote add origin https://github.com/mdnzi/FCC-Technical-Documentation-Page.git
